# Sessão 2026-05-15

## Debug — Strategy Edge Function: "resposta incompleta da IA"

**Sintoma:** usuários recebiam erro "A IA retornou uma resposta incompleta. Tente novamente em instantes." ao gerar estratégia.

**Diagnóstico via logs:**
```
stop_reason=end_turn output_tokens=1893
Claude response missing required fields: ```json { "vale_a_pena": false, ...
```
`stop_reason=end_turn` indicou que a resposta não foi truncada — o Claude terminou normalmente. O problema era que o **Haiku retornou o JSON envolto em markdown** (` ```json ... ``` `), fazendo o `JSON.parse` falhar.

**Causa raiz 1:** Claude Haiku ignora instrução "responda apenas com JSON" e adiciona code fences.
**Causa raiz 2 (secundária):** `max_tokens: 2000` era baixo demais para `step_details` longos — aumentado para 4096.

**Fixes aplicados:**
- `strategy/index.ts`: strip de code fences antes do `JSON.parse` — resiliente a reincidências
- `strategy/index.ts`: `max_tokens` 2000 → 4096
- `strategy/index.ts`: log de `stop_reason` e `output_tokens` adicionado para diagnóstico futuro
- `prompts/system.ts`: instrução explícita no topo — "FORMATO OBRIGATÓRIO: JSON puro, sem markdown, sem \`\`\`json"

**Aprendizado:** `stop_reason=end_turn` + parse falha = problema de formato. `stop_reason=max_tokens` + parse falha = limite de tokens.
